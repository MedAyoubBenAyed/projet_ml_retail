PROJECT RECAP — projet_ml_retail

1) PROJECT PURPOSE
This repository is an initial machine-learning project focused on retail customer churn analysis.
The available assets indicate a workflow for:
- exploring a prepared customer-level dataset,
- selecting/cleaning features,
- training a churn model,
- making predictions,
- exposing model behavior through an application.

Current state: the practical implementation is mostly not yet coded (several Python files are empty), but the exploration notebook and dataset are present.

3) DATASET SUMMARY
File: data/raw/retail_customers_COMPLETE_CATEGORICAL.csv
- Rows: 4372
- Columns: 52
- Target variable: Churn (binary label visible in sample rows: 0/1)

Main column groups observed:
- RFM and transaction behavior:
  Recency, Frequency, MonetaryTotal, MonetaryAvg, MonetaryStd, MonetaryMin, MonetaryMax,
  TotalQuantity, AvgQuantityPerTransaction, MinQuantity, MaxQuantity,
  AvgDaysBetweenPurchases, TotalTransactions, UniqueInvoices, AvgLinesPerInvoice, etc.
- Product/invoice diversity:
  UniqueProducts, UniqueDescriptions, ProductDiversity, BasketSizeCategory, etc.
- Customer profile and account metadata:
  Age, AgeCategory, Gender, Country, Region, RegistrationDate, AccountStatus,
  NewsletterSubscribed, LoyaltyLevel, SatisfactionScore, SupportTicketsCount, etc.
- Categorical business segments:
  RFMSegment, SpendingCategory, CustomerType, ChurnRiskCategory,
  FavoriteSeason, PreferredTimeOfDay, WeekendPreference, etc.

Notes from data sample:
- Mixed numeric and categorical features.
- Some missing values are present (e.g., Age empty for some customers).
- Multiple date formats appear in RegistrationDate.
- Some indicators of returns/cancellations exist (NegativeQuantityCount, CancelledTransactions, ReturnRatio).


4) NOTEBOOK SUMMARY (notebooks/01_data_exploration.ipynb)
The notebook is in French and contains 47 cells. It implements exploratory analysis and feature-reduction steps.

Main logical flow:
1. Imports: pandas, numpy, matplotlib, seaborn.
2. Loads dataset from ../data/raw/retail_customers_COMPLETE_CATEGORICAL.csv.
3. Basic inspection:
   - head/tail
   - shape
   - info and dtypes
   - missing values checks and percentages
4. Descriptive statistics:
   - numeric summary
   - full summary with categorical features
5. Correlation analysis on numeric columns:
   - correlation matrix
   - heatmap
   - detection of highly correlated variables (threshold around 0.8)
6. Correlation-based feature reduction:
   - identifies columns to drop from upper triangle correlation matrix
   - creates reduced dataset
7. VIF-based multicollinearity analysis:
   - uses statsmodels variance_inflation_factor
   - keeps numeric predictors and removes infinite/NaN issues before VIF
8. Additional manual feature drop appears in later cells:
   - CustomerID
   - CustomerTenureDays
   - FirstPurchaseDaysAgo

Interpretation:
- The notebook is currently the core intelligence of the repository.
- It documents feature inspection and dimensionality/multicollinearity control before modeling.
